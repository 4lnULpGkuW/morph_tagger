# MorphTagger

##### MorphTagger - модель морфологического классификатора на основе кодирующей части трансформера. Позволяет производить морфологический разбор слов на **русском языке** с высокой точностью, при этом обеспечивает молниеносную скорость обработки и низкий расход памяти.

## Структура репозитория
* **checkpoints**

    Директория сохранений парметров модели и конфигурации токенизатора.
    * Параметры модели в формате `.pt`.
    * Конфигурация токенизатора в формате `.json`.

* **data**

    Директория сохранений словарей и метрик.
    * Словари соответствия токен-индекс различных датасетов для инстанцирования обьектов класса *Vocabulary*.
    * Метрики обучения и тестирования модели.

* **dataset**

    * Датасеты для обучения и тестирования.\
    **Подразумевается, что данные в этой директории получены путем выполнения скрипта** `dataset_preprocessing.py` **и уже готовы к передаче в модель.**

* **model**

    Файлы модели
    * `model.py`  -  описание архитектуры модели.
    * `positional_encoding.py`  -  позиционное кодирование.

* **scripts**

    Вспомогательные скрипты для работы пайплайна
    * `custom_dataset.py`  -  Минимальная реализация датасета для использования с torch.utils.data.DataLoader
    * `tokenizer.py`  -  Токенизаторы.
    * `vectorizer.py`  -  Предоставляет функционал перехода от токенов к индексам, а также дополнение коротких последовательностей паддингом.
    * `vocabulary.py`  -  Класс для работы со словарем токен-индекс.

*  `dataset_creation.py`  -  Скрипт для распаковки датасетов из формата .conllu.\
Создает файлы формата .parquet, удобные для чтения и работы в pandas. Подготовленные файлы будут сохранены в той же директории, где находились .conllu файлы. Путь до датасета указывается в `.env` файле.\
**Важное ограничение**: у директорий с файлами датасетов должна быть общая родительская директория.. Аргументы командной строки:

    --dataset: ['taiga', 'syntagrus', 'merged'] - Выбор датасета для обработки.

* `dataset_preprocessing.py`  -  Скрипт для подготовки .parquet датасетов в формат, удобный для обучения и тестирования модели.\
Включает в себя токенизацию всего датасета и конвертация токенов в индексы.\
Обработанный датасет помещается в диеркторию **/dataset**. Аргументы командной строки:

    --dataset: ['taiga', 'syntagrus', 'merged'] - Выбор датасета для обработки.

    --pretrained - Если флаг указан, то будет использоваться ранее обученный токенизатор из директории **/checkpoints**

    --mfp - Целочисленное значение.\
Выбор минимальной частоты встречаемости соседних символов для их слияния в один токен при обучении токенизатора.\
Например, при MFP = 200, символы не будут обьеденены в токен, если встретились по соседству менее 200 раз.

* `test.py` - Скрипт для тестирования модели классификатора.\
Аргументы командной строки:

    user_query - Предложение для анализа.

    --morpheme - Строка с морфемами для вывода на экран. Морфемы должны быть отделены друг от друга пробелом. По умолчанию учитываются все морфемы.

    Множество доступных морфем:\
    ['upos', 'head', 'deprel', 'Mood', 'NumType', 'VerbForm', 'ExtPos', 'Reflex', 'Polarity', 'Typo', 'NameType', 'InflClass','Person',
    'Poss', 'Animacy','Degree', 'Foreign', 'Variant', 'Number', 'Gender', 'NumForm', 'Aspect', 'Case', 'PronType', 'Tense', 'Abbr', 'Voice']

    --device - Исопльзуемое устройство. default = cuda.

* `train.py` - Скрипт для обучения модели морфологического классификатора.\
Скрипт будет искать датасет в директории **/dataset**, записывать метрики обучения в **/data**, параметры модели в **/checkpoint**. Аргументы командной строки:

    --dataset: ['taiga', 'syntagrus', 'merged'] - Выбор датасета для обработки.

    --pretrained - Если флаг указан, то будет использоваться ранее обученная модель из директории **/checkpoints**.

    --batch - Размер батча для обучения. default = 96.

    --device - Исопльзуемое устройство. default = cuda.

    --checkpoint_epoch - Количество эпох для промежуточного сохранения парметров модели и метрик обучения. default = 2.

* `validate_with_dataset.py` - Скрипт для тестирования метрик модели на датасете. Скрипт будет искать датасет в директории **/dataset**, записывать метрики тестирования в **/data**. Аргументы командной строки:

    --dataset: ['taiga', 'syntagrus', 'merged'] - Выбор датасета для обработки.

    --split: ['validation', 'test'] - Выбор выборки датасета для теста. Используется либо валидационная, либо тестовая.

    --device - Исопльзуемое устройство. default = cuda.


## Использование

##### В репозитории храняться лучшие параметры модели и парметры токенизатора для этой модели. Также в директории **/dataset** уже храняться предобработанные датасеты для облегчения тестирования модели. Если необходимо обучить модель заново или воспроизвести результаты тестирования, то необходимо выполнить последовательность шагов для подготовки данных.

* Для простого тестирования на входном предложении достаточно запустить скрипт `test.py` с описанными выше аргументами. Пример запуска:

`python3 test.py "Мама мыла раму" --device cpu --morpheme "upos Animacy"`

* Для обучения модели и оценки метрик на датасете необходимо выполнить последовательность подготовительных шагов.

    1. Убедитесь, что в файле `.env` были корректно указаны пути до "сырых" датасетов формата `.conllu`.

    2. Запустите скрипт `dataset_creation.py` и передайте в аргументе короткое наименование датасета для распаковки. Пример:

        `python3 dataset_creation.py --syntagrus`

    3. Запустите скрипт `dataset_preprocessing.py` с таким же коротким наименованием датасета, что и на предыдущем шаге. Если нет необходимости обучать токенизатор заново, то можно передать параметр *--pretrained* и использовать заранее обученный токенизатор. Результатом выполнения скрипта будут подготовленные датасеты в директории **/dataset**. Пример: 

    `python3 dataset_preprocessing.py --dataset syntagrus --mfp 1000`

    или

    `python3 dataset_preprocessing.py --dataset syntagrus --pretrained`

    4. После выполнения третьего шага можно переходить к обучению или тестированию модели.

* Для оценки метрик на датасете необходимо использовать скрипт `validate_with_dataset.py`. Пример запуска:

    `python3 validate_with_dataset.py --dataset syntagrus --split test --batch 256 --device cuda`

* Для обучения модели можно воспользоваться скриптом `train.py`. Пример вызова:

    `python3 train.py --dataset merged --batch 64 --device cuda --checkpoint_epoch 5`

* Для дообучения модели достаточно дополнительно передать флаг *--pretrained*
